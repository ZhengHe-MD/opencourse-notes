1. Review

Dictionary 是一种抽象数据结构 (ADT)，维护着一个集合，我们称集合中的元素为 item，集合中的元素由键值对 (key, value) 构成，它支持以下操作：

  • insert(item)：将 item 添加到集合中
  • delete(item)：从集合中删除 item
  • search(key)：在集合中寻找 key 对应的 item，如果存在就返回

Dictionary 不维护数据的顺序关系。

Hashing from 6.006

在 6.006 中介绍的 Dictionary 实现的时空复杂度如下：

  • 时间复杂度：insert、delete 和 search 都是 O(1)O(1)O(1)
  • 空间复杂度：O(n)O(n)O(n)

具体来说，假设：

  • uuu 表示 key 的可能取值总量
  • nnn 表示在 hash table 中的 keys/items 总量
  • mmm 表示在 hash table 中的坑位 (slots) 总量

我们可以通过 hashing with chaining + table doubling 来实现 θ(1+α)\theta{(1 + \alpha)}θ(1+α) 的各操作事件复杂度，其中 α=nm\alpha = \frac{n}{m}α=mn​ 为 load factor。在推导过程中，我们做了一个很强的假设：我们使用的 hash function 满足 simple uniform hashing assumption (SUHA)，即：

Pk1≠k2{h(k1)=h(k2)}=1m\underset{k_1 \ne k_2}{\mathbb{P}}{\{h(k_1) = h(k_2)\}} = \frac{1}{m} k1​=k2​P​{h(k1​)=h(k2​)}=m1​

即 hash function 的散列过程是随机的，但我们知道 hash function 必须是稳定的，即输入相同时输出相同。随机性和稳定性是对立的，本节课的目的就是去除 simple uniform hashing 的假设，来看我们如何在理论上达到相同的时空复杂度。

2. Universal Hashing

理解 universal hashing 的直觉就是：将随机性加在 hash functions 的选择上。即从一个 hash functions 集合 H\mathcal{H}H 中随机选择一个 (记为 hhh)，其中 H\mathcal{H}H 是一个 universal hashing family，后者满足：

Ph∈H{h(k)=h(k′)}≤1mfor all k≠k′\underset{h \in \mathcal{H}}{\mathbb{P}}{\{h(k) = h(k^{'})\}} \le \frac{1}{m} for \space all \space k \ne k^{'} h∈HP​{h(k)=h(k′)}≤m1​for all k=k′

即从 H\mathcal{H}H 中随机选一个 hash function，在该情况下两个不同元素被散列到同一个位置上的概率小于 1m\frac{1}{m}m1​。到此你可能像我一样有疑问，这样随机选取 hash function 不就又丧失了稳定性吗？**如果每次散列都要从 H\mathcal{H}H 中随机取一个 hash function，那如何保证第二次遇到相同的元素还会被散列到相同的位置？**实际上，对于同一个 hash table，随机选取 hash function 的过程只会执行一次，即对于这张 table 所有 keys 都使用同一个的 hhh，直到 hash table handler 因某种条件触发 rehash 时，如 table doubling，才会重新选择 hash function。而 rehash 需要将当前所有 keys/items 取出，重新散列，详情见这里的讨论。

接下来，我们要先证明 universal hashing 能取得与 SUHA 相同的时空复杂度，然后再找到这样的 hashing family。

定理：从 uuu 中随机取任意不同的 nnn 个 key，且为每个 key 随机从某 universal hashing family H\mathcal{H}H 中取一个 hash function，记为 hhh，被散列到某一个 slot 中的 keys/items 数量预期小于 1+α1+\alpha1+α，即：

E[#keys colliding in a slot]≤1+α, where α=nm\mathbb{E}[\#keys\space colliding\space in\space a\space slot] \le 1 + \alpha,\space where\space \alpha = \frac{n}{m} E[#keys colliding in a slot]≤1+α, where α=mn​

当我们要证明某事件发生次数与某阈值的关系时，一种常用技巧就是 indicator variable。假设所有在 hash table 中的 keys 分别为 k1,k2,...,knk_1, k_2,..., k_nk1​,k2​,...,kn​，我们设 indicator variable：

Ii,j={1if h(ki)=h(kj)0otherwiseI_{i,j} = \begin{cases} 1 & if\space h(k_i) = h(k_j) \\ 0 & otherwise \end{cases} Ii,j​={10​if h(ki​)=h(kj​)otherwise​

那么：

\begin{align*} \mathbb{E}[I_{i,j}] &= \mathbb{P}{\{I_{i,j} = 1\}} \\ &= \mathbb{P}{\{h(k_i) = h(k_j)\}} \\ &\le \frac{1}{m} \space for \space any \space j \ne i \end{align*}

因此：

\begin{align*} \mathbb{E}[\#keys\space hashing\space to\space the\space same \space slot \space as \space k_i] &= \mathbb{E}[\sum_{j=i}^{n}{I_{i,j}}] \\ &= \sum_{j\ne i}^{n}\mathbb{E}[{I_{i,j}}] + \mathbb{E}[{I_{i,i}}] \\ &= \frac{n}{m} + 1 \end{align*}

以上证明了 universal hashing 能够取得与 SUHA 相同的时间复杂度。接下来举几个 universal hashing family 的例子：

Example #1: all hash functions

将所有可能的 hash functions 组成一个集合：

H={all hash functions h:{0,1,...,u−1}→{0,1,...,m−1}}\mathcal{H} = {\{all\space hash\space functions\space h: \{0,1,...,u-1\} \rightarrow \{0,1,...,m-1\}\}} H={all hash functions h:{0,1,...,u−1}→{0,1,...,m−1}}

显然，H\mathcal{H}H 本身是 universal hashing family，因为对于任意一个 key 来说，从所有可能的 hash functions 中随机取一个，该 key 被散列到某个 slot 上的概率为 1m\frac{1}{m}m1​。但 H\mathcal{H}H 只存在理论价值，不具备实践意义：

  1. 存储某个 h∈Hh \in \mathcal{H}h∈H 需要花费 log(mu)=ulog(m)≫nlog(m^u) = ulog(m) \gg nlog(mu)=ulog(m)≫n bits。而且为了存储这个 hash function，你需要使用一个 hash table 😂，这就来到了鸡生蛋还是蛋生鸡的问题
  2. 生成某个 h∈Hh \in \mathcal{H}h∈H 的时间复杂度为 Ω(u)\Omega(u)Ω(u) ，因为你需要为 uuu 个值生成对应的散列位置

Example #2: dot-product hash family

假设：

  • mmm 是一个素数
  • u=mru = m^ru=mr，其中 rrr 是一个整数

在实践中，你总能够将 mmm 和 uuu 向上取到满足上述条件的数。现在我们将所有的 keys 转化成 mmm 进制的数，即 $k = \langle k_0,k_1,…,k_{r-1} \rangle $，取某个常数 $a = \langle a_0,a_1,…,a_{r-1} \rangle $，定义 hash function：

\begin{align*} h_a{(k)} &= a \cdot k \space mod \space m \\ &= \sum_{i=0}^{r-1} a_i k_i \space mod \space m \end{align*}

于是得到 dot-product hash family H={ha∣a∈{0,1,...,u−1}}\mathcal{H} = \{h_a | a \in \{ {0, 1,..., u-1} \} \}H={ha​∣a∈{0,1,...,u−1}}。

3. Perfect Hashing

如果已知要插入 hash table 的所有 keys，数量不变，且该 hash table 只需要支持 search(key)，我们能做得更好吗？这便是 static dictionary problem。

利用 universal hashing family，我们能做到平均情况下 (average case) ，时间复杂度 O(1)O(1)O(1)，空间复杂度 O(n)O(n)O(n)，但本节要介绍的 perfect hashing 能做到最坏情况下 (worst case)，时间复杂度 O(1)O(1)O(1)，空间复杂度 O(n)O(n)O(n)，但整个 hash table 构建的时间复杂度为 polynomial (w.h.p)。
